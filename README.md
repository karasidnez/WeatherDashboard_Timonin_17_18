# Лабораторная работа №17-18. Корутины на практике: Метеосводка

 — учебное Android-приложение, демонстрирующее использование корутин Kotlin для асинхронной загрузки данных. Приложение имитирует получение погодных показателей (температура, влажность, скорость ветра) из трёх разных источников и наглядно показывает преимущества параллельного выполнения задач.
 
 ## Функциональность
- Параллельная загрузка данных с имитацией сетевых задержек
- Отображение индикаторов загрузки для каждого параметра
- Кнопка ручного обновления данных
- Имитация ошибок сети с корректной обработкой исключений
- Вычисление сводного индекса погоды с использованием `Dispatchers.Default`
- Автоматическое обновление каждые 10 секунд
- Современный интерфейс на Jetpack Compose

## Используемые технологии
- Kotlin
- Jetpack Compose
- Kotlin Coroutines
- ViewModel и StateFlow

## Контрольные вопросы
### A)В чём разница между launch и async?
`launch` запускает корутину без возврата результата (только `Job`), используется для операций, где не нужен результат.  
`async` возвращает `Deferred<T>`, позволяя получить результат через `await()`, применяется, когда результат необходим.
**Пример кода:**
```kotlin
// launch – без результата
viewModelScope.launch {
    repository.sendLog("App started")
}

// async – с результатом
val deferred = viewModelScope.async {
    repository.loadUserData()
}
val userData = deferred.await()
```
### B) Что такое suspend функция?
`suspend`-функция — это функция, которая может быть приостановлена и возобновлена без блокировки потока. Она может вызываться только из другой `suspend`-функции или корутины; из обычной функции её вызвать нельзя. Функция `delay()` внутри корутины не блокирует поток, потому что приостанавливает только текущую корутину, позволяя потоку выполнять другие задачи.

### C) Зачем нужны разные диспетчеры?
Диспетчеры определяют, в каком потоке будет выполняться код корутины, что позволяет избежать блокировки UI и оптимально использовать ресурсы устройства. Основные диспетчеры: `Main` для работы с интерфейсом, `IO` для операций ввода-вывода, `Default` для вычислительных задач. Если выполнить тяжёлое вычисление на `Dispatchers.Main`, главный поток будет заблокирован, приложение перестанет реагировать на действия пользователя и может быть закрыто системой с ошибкой ANR.

### D) Что произойдёт, если не обработать исключение в корутине?
Необработанное исключение в корутине вызовет аварийное завершение всего приложения. Для предотвращения краша необходимо перехватывать исключения с помощью конструкции `try-catch` внутри корутины или использовать `CoroutineExceptionHandler`. `try-catch` внутри `launch` позволяет локально обработать ошибку именно в этой корутине и, например, показать пользователю уведомление без падения приложения.

### E) Как работает автоматическая отмена корутин?
`viewModelScope` — это специальный `CoroutineScope`, привязанный к жизненному циклу `ViewModel`. Все корутины, запущенные в этом скоупе, автоматически отменяются при уничтожении `ViewModel` (вызов `onCleared()`), что происходит, когда пользователь закрывает экран или происходит пересоздание конфигурации. Это предотвращает утечки памяти и выполнение ненужной фоновой работы.
